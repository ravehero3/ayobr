Bug 2 — var audioDuration leaks out of scope into the progress handler 🔴

var is function-scoped, so the progress handler closure at line ~388 references audioDuration before it's assigned (it's used in the time-based fallback calculation). The progress handler runs before we reach line 420. Change to const and hoist it above the progress handler registration, or pass it in explicitly.

Bug 3 — Missing COEP/COOP headers on Render 🔴
SharedArrayBuffer (required for FFmpeg's threaded WASM) is only available when the server sends:
Cross-Origin-Embedder-Policy: require-corp
Cross-Origin-Opener-Policy: same-origin
Render doesn't add these by default. Without them, SharedArrayBuffer is undefined, FFmpeg silently falls back to single-threaded mode or fails to load entirely. Check your browser console for SharedArrayBuffer is not defined.
Fix for Render — add a _headers file in your public/ folder (if using static hosting) or configure it in render.yaml:
# render.yaml
services:
  - type: web
    headers:
      - path: /*
        name: Cross-Origin-Embedder-Policy
        value: require-corp
      - path: /*
        name: Cross-Origin-Opener-Policy
        value: same-origin
Or if you have a custom Express/Node server:
app.use((req, res, next) => {
  res.setHeader('Cross-Origin-Embedder-Policy', 'require-corp');
  res.setHeader('Cross-Origin-Opener-Policy', 'same-origin');
  next();
});
